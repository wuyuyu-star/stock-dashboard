# 美股看板开发问题记录

## 项目概述
- 技术栈：Python + Streamlit + Plotly + Pandas
- 部署：Streamlit Community Cloud（免费）
- 仓库：github.com/wuyuyu-star/stock-dashboard
- 数据源：Twelve Data（实时报价）+ yfinance（基本面/K线/财报）

---

## 问题一：yfinance 在 Streamlit Cloud 被封
**现象：** 本地运行正常，部署到 Streamlit Cloud 后报 `YFRateLimitError`  
**原因：** Streamlit Cloud 是共享服务器，大量用户共用同一批 IP，Yahoo Finance 对这些 IP 进行了封锁  
**尝试过但无效的方法：**
- curl_cffi 模拟浏览器请求
- 增加重试次数和延迟
**最终解决方案：** yfinance 仅用于基本面/K线/财报（缓存24小时，每天只请求一次，不触发限流）；实时报价改用 Twelve Data API

**值得借鉴：** 被封的不是 yfinance 这个库本身，而是"高频实时请求"这个行为。当初价格每60秒刷新一次，多用户共用 IP 导致 Yahoo 识别异常流量并封锁。最终能用回 yfinance 是因为改变了缓存策略——基本面24小时、K线1小时，频率和普通用户手动查询无异。**选 API 不只看用什么库，更要看请求频率。**

---

## 问题二：API 选型折腾
**踩坑历程：**
1. **yfinance** → Streamlit Cloud IP 被封（见问题一）
2. **FMP（Financial Modeling Prep）** → 2025年8月后免费接口全部下线
3. **Alpha Vantage** → 免费版每天只有25次请求，页面一加载就用完
4. **Twelve Data** → 免费版800次/天、8次/分钟，基本够用

**最终方案：**
- 实时报价/大盘指数 → Twelve Data（`/quote` 接口）
- K线历史数据 → yfinance（缓存1小时）
- 基本面/财报 → yfinance（缓存24小时）
- 新闻 → Twelve Data（`/news` 接口）

---

## 问题三：Twelve Data 免费版接口限制
**现象：** 基本面、评分、财报全部显示 N/A，日志无报错  
**原因：** `/statistics`、`/income_statement`、`/balance_sheet` 接口需要付费套餐（Pro $29/月起），免费版返回 403 错误，被代码静默忽略  
**排查方法：** 在页面加入调试输出块 `st.expander` 显示原始 API 返回，发现 `{"code":403, "status":"error"}`  
**解决方案：** 改用 yfinance 获取上述数据，绕过 Twelve Data 付费限制

---

## 问题四：Twelve Data 频率限制（8次/分钟）
**现象：** 搜索某些股票报"未找到股票代码"，换一只又正常  
**原因：** 页面加载时同时请求4个大盘指数 + 自选股 + 当前股票，超过8次/分钟上限  
**解决方案：**
- 大盘指数单独用5分钟缓存的函数 `fetch_quote_index()`，避免频繁刷新
- 个股报价60秒缓存
- K线、基本面全部迁移到 yfinance，减少 Twelve Data 请求量

## 问题五：API Key 安全
**事故：** 用户在聊天中直接粘贴了 GitHub Token 和多个 API Key  
**风险评估：**
- GitHub Token：高风险，可推送/删除代码，需立即吊销重新生成
- Twelve Data Key：低风险，只能读取公开股票数据，但建议更换
- FMP Key：低风险，同上
**正确做法：**
- API Key 存在 Streamlit Cloud → Settings → Secrets
- 代码中用 `st.secrets["KEY_NAME"]` 读取
- 永远不要在代码文件或聊天中直接写 Key

## 问题六：语法错误
**现象1：** `SyntaxError: closing parenthesis '}' does not match '('`  
**位置：** 评分 tab 的 gauge 图表  
**原因：** `go.Figure(go.Indicator(...))` 括号层级写错，少了一个 `)`  
**正确写法：**
```python
fig = go.Figure(go.Indicator(
    mode="gauge+number", value=score,
    gauge={...}
))  # 注意两个右括号
```

**现象2：** `StreamlitDuplicateElementId`  
**原因：** 两个 gauge 图表没有设置不同的 key  
**解决：** `st.plotly_chart(fig, key="gauge_valuation")` 和 `key="gauge_health"`

## 问题七：DataFrame 操作错误
**现象：** 财报 tab 列名找不到（KeyError）  
**原因：** yfinance 返回的 DataFrame 列名是英文（`Total Revenue`、`Net Income`），而代码用的是小写下划线格式（`revenue`、`net_income`）  
**解决：** 列名映射表改为匹配 yfinance 实际返回的列名：
```python
inc_map = {
    "fiscal_date": "年度",
    "Total Revenue": "营收",
    "Gross Profit": "毛利润",
    "Operating Income": "营业利润",
    "Net Income": "净利润"
}
```

## 问题八：52周区间 AttributeError
**现象：** `AttributeError` on line `c3.metric("52周区间", f"${...}")`  
**原因：** f-string 里直接调用 `.get()` 嵌套写法在某些情况下报错  
**解决：** 提前赋值再使用：
```python
_52w = quote.get("fifty_two_week", {})
_52w = _52w if isinstance(_52w, dict) else {}
c3.metric("52周区间", f"${_52w.get('low','?')} ~ ${_52w.get('high','?')}")
```

## 问题九：股票代码失效
**现象：** 搜索 TWTR 报"未找到股票代码"  
**原因：** Twitter 2022年被马斯克私有化退市，TWTR 代码已不存在  
**解决：** 同行对比列表中将 TWTR 替换为 RDDT（Reddit）

## 问题十：虚拟环境膨胀
**现象：** stock_env 占用 511MB  
**原因：** 开发过程中安装了 yfinance、curl_cffi 及其依赖（multitasking、GitPython 等），后来虽然弃用但残留  
**解决：** 删除旧环境重建：
```bash
rm -rf ~/claudecode专用/stock_env
python3 -m venv stock_env
source stock_env/bin/activate
pip install streamlit plotly pandas requests urllib3 openpyxl yfinance
```
